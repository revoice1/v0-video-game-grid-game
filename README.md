# v0-video-game-grid-game

A daily video game trivia grid game (think Immaculate Grid, but for games). Players fill a 3×3 grid where each cell must satisfy both its row and column category — e.g. a game that is both a PlayStation exclusive and an RPG.

This is a [Next.js](https://nextjs.org) project bootstrapped with [v0](https://v0.app).

[Continue working on v0 ->](https://v0.app/chat/projects/prj_hPb9BuYVa2rCYaTFQe7J6RVtn7uK)

## Getting Started

Create a `.env.local` file from `.env.example`, then:

```bash
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000).

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | Supabase anon key |
| `TWITCH_IGDB_CLIENT_ID` | ✅ | IGDB API client ID (via Twitch dev console) |
| `TWITCH_IGDB_CLIENT_SECRET` | ✅ | IGDB API client secret |
| `PUZZLE_MIN_VALID_OPTIONS` | — | Minimum valid answers per cell (default: `3`) |
| `PUZZLE_GENERATION_MAX_ATTEMPTS` | — | Max candidate grids to try before failing (default: `12`) |
| `PUZZLE_VALIDATION_SAMPLE_SIZE` | — | IGDB matches sampled when validating each cell (default: `40`) |

## Architecture

### API Routes

| Route | Description |
|-------|-------------|
| `GET /api/puzzle?mode=daily\|practice` | Returns today's puzzle as JSON. Reads from DB if already generated; otherwise generates, stores, and returns. |
| `GET /api/puzzle-stream?mode=daily\|practice` | Same but streams real-time generation progress as Server-Sent Events. Used by the game client for live progress bars. |
| `POST /api/guess` | Validates a game guess against a cell's row/column categories. |
| `GET /api/search?q=...` | Searches IGDB for games matching a query. |
| `POST /api/stats` | Records a completed game session. |

### Puzzle Generation

Puzzles are generated by sampling random category combinations from IGDB (platforms, genres, decades, game modes, themes, perspectives) and validating that each of the 9 cells has at least `PUZZLE_MIN_VALID_OPTIONS` valid answers.

**Daily puzzles** are generated once and stored in the `puzzles` table including their `cell_metadata` (per-cell valid answer counts). Subsequent requests for the same day read directly from the DB — no IGDB calls needed.

**Practice puzzles** are generated fresh on each request and are not cached between sessions.

### SSE Progress Stream

`/api/puzzle-stream` keeps the HTTP connection open and emits events as generation progresses:

```
data: {"type":"progress","pct":12,"message":"Attempt 1/12: picking categories..."}
data: {"type":"progress","pct":47,"message":"Attempt 1/12: checking intersection 5/9..."}
data: {"type":"puzzle","puzzle":{...}}   ← final event, connection closes
```

The client (`components/game/game-client.tsx`) opens this with `EventSource` and drives the progress bar from real server events rather than a fake timer.

### Session State

Game state (guesses, remaining attempts, completion) is stored in `localStorage` keyed by puzzle ID, so returning users resume where they left off without needing an account.
